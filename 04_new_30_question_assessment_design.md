# 30 题人格娱乐评估系统设计草案

> 定位：娱乐型人格画像系统，不用于心理诊断、职业评估或严肃决策。
> 目标：30 题内完成，结果要像自己、能解释、适合分享。

---

## 1. 设计原则

1. **30 题固定长度**：减少用户完成压力，控制在 3-5 分钟内。
2. **10 个维度，每维 3 题**：比 15 维 × 2 题更稳，单题误选影响更小。
3. **场景题优先**：少问抽象性格，多问真实反应。
4. **四选一强制取向**：不设置“都可以”逃生门，降低中庸答案比例。
5. **算法解释清楚**：先算维度分，再匹配人格模板，不做纯玄学跳转。
6. **文案允许夸张**：结果是娱乐产品，表达要有记忆点，但避免真实伤害。

---

## 2. 维度体系

| 编号 | 维度 | 高分含义 | 低分含义 |
|---|---|---|---|

| D2 | 推进力 | 目标导向、主动推进、讨厌悬而未决 | 容易观望、拖延、低能量启动 |
| D3 | 风险偏好 | 敢试、敢冲、容忍不确定 | 谨慎、防御、先避坑 |
| D4 | 情绪浓度 | 情绪反应强、表达浓烈 | 情绪低显性、冷处理、延迟反应 |
| D5 | 亲密投入 | 关系里认真、投入、容易绑定 | 保留后手、投入克制 |
| D6 | 边界需求 | 需要空间、独处、清晰边界 | 喜欢融合、陪伴、强连接 |

## 3. 计分方式

每题 4 个选项，分值为 `1 / 2 / 3 / 4`。

每个维度 3 题，原始分范围：

```text
raw = q1 + q2 + q3
min = 3
max = 12
```

归一化为 0-100：

```javascript
score = Math.round(((raw - 3) / 9) * 100)
```

维度等级：

```javascript
if (score <= 33) return 'L'
if (score <= 66) return 'M'
return 'H'
```

人格匹配：

```javascript
// 每个人格有 10 维目标分，例如 [90, 85, 35, ...]
// 每个维度可有权重，核心维度权重更高
distance = sum(abs(userScore[i] - typeTarget[i]) * weight[i])
similarity = Math.round(100 - distance / maxDistance * 100)
```

建议每个人格设置：

- `target`: 10 维目标分
- `weights`: 10 维权重，核心维度为 `1.5`，普通维度为 `1`
- `must`: 可选硬条件，例如控制感必须高于 70，否则不能成为 `总控台`

## 7. 结果页结构

每个结果建议包含：

1. 人格名：例如 `CTRL 总控台`
2. 一句话暴击：适合截图传播
3. 匹配度：`87%`
4. 三个关键词：例如 `控场 / 推进 / 边界`
5. 核心机制：为什么你会这样
6. 优势：你在哪些场景很强
7. 翻车点：你最容易怎么失控
8. 关系模式：亲密关系/朋友关系中的表现
9. 工作模式：项目、协作、压力下的表现
10. 相邻人格：Top 2 / Top 3，降低单一结果的绝对感

## 8. 与当前项目的接入方式

建议分两步做：

1. **保留现有 SBTI 作为 v1**
   - 不破坏当前页面和题库。
   - 新系统作为 `v2` 数据结构单独引入。

2. **新增测试模式切换**
   - 首页增加 `经典 SBTI` / `新 30 题人格系统` 两个入口。
   - 共用答题 UI、结果页 UI、分享能力。
   - `questionsV1 / typesV1 / scoringV1` 与 `questionsV2 / typesV2 / scoringV2` 分离。

推荐文件拆分：

```text
index.html
data/
  sbti-v1.js
  personality-v2.js
```

如果继续保持单文件，也可以先用：

```javascript
const ASSESSMENTS = {
  sbti: { questions, types, scoring },
  new30: { questions: questionsV2, types: typesV2, scoring: scoringV2 }
};
```
